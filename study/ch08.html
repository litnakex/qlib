CHAPTER 08

综合实战：端到端跑通一个量化流程

前七章我们把 Qlib 拆成了一个个零件：数据、处理器、模型、策略、回测、工作流。这一章把它们重新拼成一台完整的机器——从零数据到一份带收益曲线的实验报告，走完整整一圈。

1. 先看全景：一次完整流程长什么样

想象你要开一家餐厅。前七章你分别学会了「进货（数据层）」「洗菜切配（处理器）」「厨师（模型）」「上菜顺序（策略）」「模拟营业算账（回测）」「记账本（工作流）」。现在，是时候把它们串成一条从食材到账单的完整流水线了。

  下载数据        构建模型/数据集       训练         预测&分析          回测          看报告
 ┌────────┐     ┌──────────────┐   ┌───────┐   ┌────────────┐   ┌──────────┐   ┌────────┐
 │GetData │ ──▶ │init_instance │──▶│ model │──▶│SignalRecord│──▶│PortAna   │──▶│ mlruns │
 │qlib_data│    │ _by_config   │   │ .fit  │   │SigAnaRecord│   │Record    │   │ 结果   │
 └────────┘     └──────────────┘   └───────┘   └────────────┘   └──────────┘   └────────┘
   第2章           第4/5章           第5章        第4/7章           第6章          第7章

Qlib 提供两条等价的路径来走完这条流水线：

  • 路径 A（配置驱动 / qrun）：把整条流水线写进一个 YAML 文件，一行命令跑完。适合复现基准、批量实验。
  • 路径 B（代码驱动 / Python）：用 Python 一步步搭积木，每一步都看得见、改得动。适合调试、二次开发、理解内部机制。
提示两条路径底层调用完全相同的组件，产出的结果也一致。官方脚本 examples/workflow_by_code.py:8 的注释就明确写着：这个脚本「nearly does the same thing as qrun XXX.yaml」。

2. 路径 A：配置驱动，一行命令跑完

这是最快的上手方式。三步走：装环境 → 下数据 → 跑配置。

第一步：确保数据到位

# 安装 qlib（若还没装，编译型 Cython 算子会一并构建）
pip install pyqlib

# 下载中国 A 股日频数据到默认目录 ~/.qlib/qlib_data/cn_data
python -m qlib.run.get_data qlib_data \
    --target_dir ~/.qlib/qlib_data/cn_data \
    --region cn

第二步：用 qrun 跑 LightGBM + Alpha158 基准

qrun 是 Qlib 自带的命令行入口，参数就是一个 YAML 配置文件的路径：

# 进入仓库目录后执行
cd examples
qrun benchmarks/LightGBM/workflow_config_lightgbm_Alpha158.yaml

这个 YAML（见 examples/benchmarks/LightGBM/workflow_config_lightgbm_Alpha158.yaml）里其实就藏着整条流水线。拆开看，它的顶层字段和前面章节一一对应：

qlib_init:                # 第2章：初始化，指定数据目录和地区
    provider_uri: "~/.qlib/qlib_data/cn_data"
    region: cn
market: &market csi300     # 股票池：沪深300
benchmark: &benchmark SH000300  # 业绩基准：沪深300指数
task:
    model:                 # 第5章：模型 = LightGBM (LGBModel)
        class: LGBModel
        module_path: qlib.contrib.model.gbdt
    dataset:               # 第4章：DatasetH + Alpha158 处理器
        class: DatasetH
        kwargs:
            handler:
                class: Alpha158
            segments:      # 训练/验证/测试三段时间切分
                train: [2008-01-01, 2014-12-31]
                valid: [2015-01-01, 2016-12-31]
                test:  [2017-01-01, 2020-08-01]
    record:                # 第7章：三个 Record 依次生成信号、分析、回测
        - class: SignalRecord
        - class: SigAnaRecord
        - class: PortAnaRecord
提示YAML 里的 &market / *market 是 YAML 的锚点语法（anchor / alias），意思是「定义一次、多处引用」，避免重复写 csi300。

第三步：查看结果

跑完后，所有产物被记录进当前目录下的 mlruns/ 文件夹（这正是第 7 章讲的 MLflow 后端）。启动 UI 浏览：

mlflow ui   # 默认在 http://127.0.0.1:5000 打开

你会在其中看到实验名、参数、IC/ICIR 等指标，以及回测生成的收益曲线图片和 .pkl 结果文件。

3. 路径 B：代码驱动，逐段拆解 workflow_by_code.py

YAML 太「黑箱」？我们把官方脚本 examples/workflow_by_code.py 一段段拆开，你会发现每一步都对应前面某一章的知识点——这就是本教程的「知识回环」。

3.1 初始化：init（对应第 2 章）

import qlib
from qlib.constant import REG_CN
from qlib.tests.data import GetData

provider_uri = "~/.qlib/qlib_data/cn_data"
GetData().qlib_data(target_dir=provider_uri, region=REG_CN, exists_skip=True)
qlib.init(provider_uri=provider_uri, region=REG_CN)

GetData().qlib_data(...) 若数据已存在会跳过下载（exists_skip=True）。qlib.init 是使用 Qlib 前的必做步骤：它告诉 Qlib 去哪读数据、按哪个市场的交易规则运行（REG_CN = 中国 A 股）。

3.2 搭积木：init_instance_by_config（对应第 4、5 章）

from qlib.utils import init_instance_by_config
from qlib.tests.config import CSI300_GBDT_TASK

model = init_instance_by_config(CSI300_GBDT_TASK["model"])
dataset = init_instance_by_config(CSI300_GBDT_TASK["dataset"])

这是 Qlib 最核心的「魔法」：init_instance_by_config 读一个含 class / module_path / kwargs 的字典，动态 import 并实例化对象。CSI300_GBDT_TASK 定义在 qlib/tests/config.py:124，它的 model 就是 LightGBM，dataset 就是 DatasetH 套 Alpha158——和路径 A 的 YAML 完全等价。这也解释了 YAML 为什么长那样：它就是这个字典的文本形式。

3.3 配置回测：port_analysis_config（对应第 6 章）

port_analysis_config = {
    "executor": {  # 执行器：逐日模拟
        "class": "SimulatorExecutor",
        "kwargs": {"time_per_step": "day", "generate_portfolio_metrics": True},
    },
    "strategy": {  # 策略：每天持有打分最高的 topk 只，换掉 n_drop 只
        "class": "TopkDropoutStrategy",
        "kwargs": {"signal": (model, dataset), "topk": 50, "n_drop": 5},
    },
    "backtest": {  # 回测环境：起止时间、初始资金、交易成本
        "start_time": "2017-01-01", "end_time": "2020-08-01",
        "account": 100000000,
        "exchange_kwargs": {"limit_threshold": 0.095, "deal_price": "close",
            "open_cost": 0.0005, "close_cost": 0.0015, "min_cost": 5},
    },
}

这里 topk=50, n_drop=5 表示：每天持仓 50 只股票，剔除掉分数掉出前列的 5 只、再买入 5 只新的。limit_threshold=0.095 模拟 A 股涨跌停（±9.5% 附近不能成交）。这些正是第 6 章的策略与交易成本参数。

3.4 启动实验 + 训练 + 三段记录（对应第 5、7 章）

from qlib.workflow import R
from qlib.workflow.record_temp import SignalRecord, SigAnaRecord, PortAnaRecord
from qlib.utils import flatten_dict

with R.start(experiment_name="workflow"):     # ① 开一个实验上下文
    R.log_params(**flatten_dict(CSI300_GBDT_TASK))  # ② 记录超参
    model.fit(dataset)                          # ③ 训练模型（第5章）
    R.save_objects(**{"params.pkl": model})   # ④ 保存模型

    recorder = R.get_recorder()                 # ⑤ 拿到本次记录器
    sr = SignalRecord(model, dataset, recorder) # ⑥ 生成预测信号
    sr.generate()

    sar = SigAnaRecord(recorder)                # ⑦ 信号分析：IC/ICIR
    sar.generate()

    par = PortAnaRecord(recorder, port_analysis_config, "day")  # ⑧ 组合回测
    par.generate()

逐行对照知识点：

  • ① R.start：第 7 章的实验管理入口，with 块内的一切都会被记录进 mlruns，异常也会被妥善收尾。
  • ③ model.fit(dataset)：第 5 章。模型从 dataset 的 train/valid 段取特征与标签开始学习。
  • ⑥ SignalRecord：对 test 段做预测，把每只股票每天的打分（signal）存下来。
  • ⑦ SigAnaRecord：只看预测分数与真实收益的相关性，产出 IC、ICIR、Rank IC 等（见 qlib/workflow/record_temp.py:265）。它不涉及交易，是纯粹的信号质量体检。
  • ⑧ PortAnaRecord：把信号交给策略去真实「下单」，跑回测，产出年化收益、最大回撤、信息比率等（第 6 章）。
提示脚本里还有一行 example_df = dataset.prepare("train")（workflow_by_code.py:64），它演示了 dataset 可以独立取数据、脱离训练单独使用——调试特征时非常有用。

4. 读懂输出：这些指标到底在说什么

跑完后你会看到一堆指标。别被术语吓到，用大白话理解：

指标通俗解释看什么
IC
(信息系数)		模型「打分」和「第二天实际涨跌」之间的相关系数。像考试预测准不准。越高越好；日频 A 股能到 0.03~0.05 已算不错。
Rank IC用排名算的 IC，只关心「谁比谁强」的顺序对不对，对极端值更稳健。选股更看重它。
ICIRIC 的均值 ÷ 标准差（ic.mean()/ic.std()）。衡量预测能力「稳不稳」，不只是「准不准」。越高说明模型发挥稳定。
年化收益
(annualized_return)		把这段回测的收益折算成「一年能赚百分之几」。要和基准（沪深300）对比才有意义。
最大回撤
(max_drawdown)		从最高点跌到最低点最惨亏了多少。衡量「最难受时有多难受」。负值，绝对值越小越好。
信息比率
(information_ratio)		相对基准的超额收益 ÷ 超额收益的波动。衡量「性价比」——每承担一份风险换来多少超额收益。越高越好，>0.5 通常不错。
注意IC 高不等于赚钱！IC 只体现「信号质量」；能不能转化成收益，还取决于策略、换手成本、涨跌停约束。所以 Qlib 才要 SigAnaRecord（看信号）和 PortAnaRecord（看真金白银）两套报告一起看。

6. 下一步学什么

跑通一条流水线只是起点。当你想让模型「活」起来、跟上市场变化，可以顺着这些方向深入，每个方向仓库里都有现成示例：

  • Rolling 滚动训练：市场会变，定期用最新数据重训模型。参考 examples/model_rolling/ 与 examples/benchmarks_dynamic/。
  • 在线服务（Online Serving）：把模型部署成每日自动更新预测的服务。参考 examples/online_srv/。
  • 强化学习（RL）：用 RL 做订单执行、择时。参考 examples/rl/ 与 examples/rl_order_execution/。
  • 嵌套决策执行：日频选股 + 日内高频执行的多层框架。参考 examples/nested_decision_execution/。
  • Meta-Learning / 超参搜索：让模型学会「如何学习」，或自动调参。参考 examples/hyperparameter/。
  • 自定义因子与模型：写自己的 Alpha 表达式、接入自研模型——回到第 4、5 章，配合 qlib/contrib/data/handler.py 和 qlib/contrib/model/ 自行扩展。
  • 高频数据：分钟级/orderbook 数据处理。参考 examples/highfreq/ 与 examples/orderbook_data/。
提示无论走哪个方向，套路都一样：换掉流水线里的某个「零件」（换个模型、换个策略、加一层 rolling），其余骨架不变。这正是 Qlib 模块化设计的价值——你现在已经掌握了这套骨架。

📝 课后练习题

1.（概念题）路径 A（qrun + YAML）和路径 B（Python 脚本）有什么关系？为什么说它们「几乎等价」？
参考答案
两者底层调用完全相同的组件。YAML 里的 class/module_path/kwargs 结构，正是 init_instance_by_config 所读取的字典的文本形式；qrun 内部就是解析 YAML 后调用同样的 init → fit → 三个 Record 流程。所以产出结果一致。区别在于：YAML 适合快速复现和批量实验，Python 脚本适合调试与二次开发。
2.（概念题）SigAnaRecord 和 PortAnaRecord 分别回答什么问题？为什么两个都要看？
参考答案
SigAnaRecord 只评估「信号质量」，产出 IC/ICIR/Rank IC，衡量预测分数与真实收益的相关性，不涉及交易。PortAnaRecord 把信号交给策略真实回测，产出年化收益、最大回撤、信息比率等真金白银的指标。两个都要看是因为：IC 高不代表能赚钱——收益还受策略、换手成本、涨跌停等约束影响。
3.（代码填空）补全启动实验并训练的关键三行（开实验上下文、训练、生成信号）。 
with R.____(experiment_name="workflow"):
    model.____(dataset)
    sr = ____(model, dataset, R.get_recorder())
    sr.generate()
参考答案
依次为：R.start(...)、model.fit(dataset)、SignalRecord(model, dataset, R.get_recorder())。
4.（改错题）某同学在 macOS 上运行脚本直接闪退（segfault），且他从没调用过 qlib.init，代码第一行就是 dataset.prepare("train")。请指出两处问题。
参考答案
(1) 任何数据操作前必须先 qlib.init(provider_uri=..., region=REG_CN)，否则报未初始化错误。(2) macOS 的 segfault 通常是 OpenMP 库冲突，设置 export KMP_DUPLICATE_LIB_OK=TRUE（或减小模型 num_threads）可缓解。此外还需确保已下载数据。
5.（思考题）你把 topk 从 50 改成 5、n_drop 保持 5，回测收益波动骤然变大。为什么？
参考答案
组合只持有 5 只股票时，单只股票的涨跌对总收益影响巨大，分散化不足，波动（和最大回撤）自然放大；同时 n_drop=5 意味着每天几乎把整个组合换一遍，换手率和交易成本也急剧上升。应保证 n_drop 远小于 topk 以维持稳定的分散组合。
6.（思考题）你的模型上线三个月后表现明显下滑。属于本章「下一步」的哪个方向？大致怎么做？
参考答案
这是市场分布漂移问题，对应 Rolling 滚动训练 / 在线服务方向。做法：定期（如每月/每季）用最新数据重新训练模型并更新预测，而非用一个一训永逸的静态模型。参考 examples/model_rolling/ 和 examples/online_srv/。
