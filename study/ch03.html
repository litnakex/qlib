CHAPTER 03

数据层 Data Layer

Qlib 最独特、也最容易被低估的部分就是它的数据层：一个能把 $close、Ref($close,1)、Mean($close,20) 这样的「公式」直接当成数据来读取的表达式引擎。本章带你看清 D 接口、算子系统、内存缓存 H、磁盘 .bin 存储和点数据 PIT 这五块拼图。

1. 为什么需要一个「数据层」

先想象一个场景。你是量化研究员，手头有 5000 只股票、10 年的日线数据。你想算一个因子：过去 20 天收盘价均线相对当前价格的偏离。用传统方式，你要：读 CSV → 转成 DataFrame → groupby 股票 → rolling(20).mean() → 对齐日期 → 处理停牌缺失……每换一个因子，这套流程都要重写一遍，而且极易在「对齐日期」这一步偷偷引入 bug。

Qlib 的思路完全不同：把「因子」表达成一个字符串公式，交给数据层去算。你只写 "$close/Mean($close,20)-1"，数据层负责读原始数据、滚动计算、缓存、对齐，最后返回一个规整的 DataFrame。研究员只关心「因子长什么样」，不关心「数据怎么搬」。

提示 数据层的对外统一入口是全局对象 D（definition 见 qlib/data/data.py:1289，它是一个 Wrapper，在 qlib.init() 时被绑定到具体的 LocalProvider）。你几乎所有取数操作都从 from qlib.data import D 开始。 
   你的因子公式 "$close/Mean($close,20)-1"
            │
            ▼
   ┌──────────────────────────┐
   │   D  (BaseProvider 接口)  │  calendar / instruments / features
   └──────────────────────────┘
            │  按职责拆分给各 Provider
   ┌──────────┬───────────┬──────────────┐
   ▼          ▼           ▼              ▼
 Cal        Inst      ExpressionD     DatasetD   (data.py 中的 Wrapper)
(日历)     (股票池)   (算子求值)      (拼装成表)
            │
            ▼
   内存缓存 H  ──miss──▶  磁盘 .bin 存储 (storage/)

2. D 接口：calendar / instruments / features

D 的方法都定义在 BaseProvider（qlib/data/data.py:1140）里，最常用的三个是：

  • D.calendar(start_time, end_time, freq) — 取交易日历（一串 Timestamp）。
  • D.instruments(market) / D.list_instruments(...) — 取股票池（market 如 "csi300"、"all"）。
  • D.features(instruments, fields, start_time, end_time, freq) — 核心，把一批字段公式算成一张宽表。

先做基本初始化，然后取日历和股票池：

import qlib
from qlib.data import D
qlib.init(provider_uri="~/.qlib/qlib_data/cn_data", region="cn")

# 交易日历：返回一个 Timestamp 列表
cal = D.calendar(start_time="2020-01-01", end_time="2020-01-10", freq="day")
print(cal[:3])
# [Timestamp('2020-01-02'), Timestamp('2020-01-03'), Timestamp('2020-01-06')]

# 股票池：返回一个 config dict（market + filter_pipe）
inst = D.instruments(market="csi300")
# 列出某时间段内实际在池的成分股
codes = D.list_instruments(inst, start_time="2020-01-01",
                           end_time="2020-01-31", as_list=True)

重头戏是 D.features。fields 是一批表达式字符串，返回一个以 (instrument, datetime) 为双层索引的 DataFrame，列名就是你传入的公式：

# 取平安银行的收盘价、昨收、20 日均线
df = D.features(
    ["SH600000"],
    ["$close", "Ref($close, 1)", "Mean($close, 20)"],
    start_time="2020-01-01",
    end_time="2020-01-31",
    freq="day",
)
print(df.head())
#                        $close  Ref($close, 1)  Mean($close, 20)
# instrument datetime
# SH600000   2020-01-02   12.30           12.10             11.95
#            2020-01-03   12.42           12.30             11.99
# ...
要点 D.features 内部会调用 DatasetD.dataset(...)（见 data.py:1186），并根据 disk_cache 参数决定跳过(0)/使用(1)/替换(2)磁盘缓存。字段前的 $ 表示这是一个原始行情字段（Feature），如 $close、$open、$volume、$factor（复权因子）。
陷阱 返回的 DataFrame 是双层 MultiIndex（instrument 在外、datetime 在内）。很多新手直接 df["$close"].plot() 会因为把多只股票混在一条曲线里而画出锯齿。要么先 df.loc["SH600000"] 切出单只，要么 unstack(level=0)。

3. 表达式算子系统：因子就是一棵表达式树

为什么 "Mean($close,20)" 这样一个字符串能被计算？因为 Qlib 把它解析成一棵表达式树：$close 是叶子节点（Feature），Mean 是一个作用在它上面的算子节点，20 是窗口参数。求值时从叶子往上递归计算。

所有算子都定义在 qlib/data/ops.py，按输入个数与形态分成几大家族（基类见 qlib/data/base.py 的 Expression / ExpressionOps）：

  • ElemOperator（元素级，单输入）：Abs、Log、Sign、Not —— 逐元素变换（ops.py:37）。
  • PairOperator（双输入）：Add、Sub、Mul、Div、Greater、Gt/Lt/Eq 等 —— 两个表达式做二元运算（ops.py:231）。
  • Rolling（滚动窗口，单输入 + 窗口 N）：Ref、Mean、Sum、Std 等 —— 最常用的一类（ops.py:713）。
  • PairRolling（双输入滚动）：Corr、Cov（ops.py:1387）。

下面是 ops.py 中真实存在的常用算子速查表：

算子类别含义示例
Ref(x, N)RollingN 期前的值（N=0 取首值，N<0 是未来）Ref($close, 1) 昨收
Mean(x, N)RollingN 日滚动均值（MA）Mean($close, 20)
Sum(x, N)RollingN 日滚动求和Sum($volume, 5)
Std(x, N)RollingN 日滚动标准差Std($close, 20)
Max/Min(x, N)RollingN 日最大/最小值Max($high, 60)
Delta(x, N)Rolling当前值减 N 期前的值Delta($close, 5)
Rank(x, N)Rolling当前值在窗口内的百分位排名Rank($close, 20)
Slope(x, N)Rolling对时间做线性回归的斜率Slope($close, 10)
Corr(x, y, N)PairRolling两序列 N 日滚动相关系数Corr($close, $volume, 10)
Add/Sub/Mul/DivPair四则运算$high - $low
Abs/Log/SignElem绝对值/对数/符号Log($volume)

这些算子可以任意嵌套组合成复杂因子。比如「20 日动量剔除均值」：

fields = [
    "$close / Ref($close, 20) - 1",          # 20 日收益率
    "($close - Mean($close, 20)) / Std($close, 20)",  # z-score
    "Corr($close, Log($volume+1), 10)",       # 量价相关
]
df = D.features(["SH600000"], fields, "2020-01-01", "2020-12-31")
性能 滚动类算子（如 Slope/Rsquare/Resi）的核心循环由 Cython 实现，见 ops.py:18-19 从 ._libs.rolling / ._libs.expanding 导入的 rolling_slope 等函数。如果你在仓库根目录直接 import qlib 会报 ImportError，因为 Cython 扩展还没编译——这就是 ops.py:20-24 那段友好报错的用意。安装用 pip install . 或 python setup.py build_ext --inplace 编译。

4. 内存缓存 H：同一个因子别算两遍

研究过程中你会反复取同一批数据（改改模型、调调参数，但特征不变）。每次都从磁盘读、重算 rolling，太浪费。Qlib 用一个全局单例 H（H = MemCache()，见 qlib/data/cache.py:1199）在进程内存里缓存结果。

MemCache（cache.py:137）内部分成三格，用短键取用：

  • H["c"] — 日历缓存（calendar）
  • H["i"] — 股票池缓存（instrument）
  • H["f"] — 特征/表达式缓存（feature）

每格是一个带容量上限的 MemCacheUnit，超限时按 LRU 淘汰（limit_type 可选 "length" 按条数或 "sizeof" 按字节，见 cache.py:121-135）。

易踩坑 qlib.init() 里会调用 H.clear()（见 qlib/__init__.py:56）清空所有内存缓存。所以：如果你在同一个进程里重新 init（比如换了 provider_uri），之前缓存的数据会全部失效并重新读取。这解释了为什么有人反映「第二次 init 后第一条 features 又变慢了」——不是 bug，是缓存被主动清了。想手动清也可以直接 H.clear()。 
from qlib.data.cache import H
print(len(H["f"]))   # 当前特征缓存里有多少条
H.clear()          # 手动清空三格缓存

5. 磁盘存储：.bin 格式与 storage 模块

内存缓存 miss 时，数据从磁盘读。Qlib 不用 CSV/Parquet，而是自定义的紧凑二进制 .bin 格式，配合 qlib/data/storage/ 下的存储后端。抽象接口在 storage.py，本地文件实现在 file_storage.py，分三类：

  • CalendarStorage — 交易日历，一个市场一份，存所有交易日。
  • InstrumentStorage — 股票池，记录每只股票的 (code, start, end) 在池区间。
  • FeatureStorage（FileFeatureStorage，file_storage.py:285）— 行情特征，每只股票每个字段各一个 .bin 文件。

路径规则一目了然（file_storage.py:289）：

# 文件名 = {股票代码小写}/{字段小写}.{频率}.bin
self.file_name = f"{instrument.lower()}/{field.lower()}.{freq.lower()}.bin"

~/.qlib/qlib_data/cn_data/
├── calendars/
│   └── day.txt              # 所有交易日
├── instruments/
│   ├── all.txt              # 全市场股票池
│   └── csi300.txt           # 沪深300 成分（含在池区间）
└── features/
    ├── sh600000/
    │   ├── close.day.bin    # 平安银行日频收盘价
    │   ├── open.day.bin
    │   └── volume.day.bin
    └── sh600519/
        └── close.day.bin

每个 .bin 文件头部存一个起始索引（对应日历下标），后面紧跟一段 float32 数组。FileFeatureStorage.data（file_storage.py:296）返回一个以日历下标为 index 的 pd.Series；__getitem__（file_storage.py:346）支持按下标或切片读取。这种「一字段一文件 + 定长数组」的设计让按列读取和滚动计算都非常快。

提示 正因为特征是按列（字段）分文件存的，Qlib 取 $close 时完全不用碰 $volume 文件——这是典型的列式存储优势。想加一个新字段（比如自定义因子），只需在对应股票目录下补一个 .bin 即可，无需重写整表。

6. 点数据 PIT：拒绝「未来函数」

前面讲的都是行情数据（价格、成交量），这类数据当天就产生、当天就知道。但财务数据（营收、净利润、ROE）不一样——它有「发生的季度」和「公布的日期」两个时间，而且往往滞后一两个月。

举个通俗例子。某公司 2020 年一季度（Q1，对应 3 月 31 日）的净利润，实际上是在 2020 年 4 月底才发布公告。如果你在回测里于 2020 年 4 月 1 日就用上了 Q1 的净利润，你就"提前"知道了还没公布的数据——这就是未来函数（look-ahead bias）。

危险 未来函数是量化里最隐蔽、也最致命的错误。用了它，回测收益曲线会漂亮到令人窒息，因为策略"偷看"了未来。可一旦实盘，未来信息拿不到，策略立刻失效。它不会报错、不会崩溃，只会让你误以为找到了圣杯——直到亏钱为止。

Qlib 用 PIT（Point-In-Time，时点数据）解决这个问题：财务数据不仅记录「属于哪个报告期」，还记录「在哪天可被市场知道」。取数时，对每个交易日只返回截至当天已经公布的那期财报，从根本上杜绝偷看未来。

PIT 的算子实现在 qlib/data/pit.py，核心是两个类：

  • P（pit.py:24，继承自 ElemOperator）— 加载某个 PIT 字段，按 cur_time（当前交易日）取"当时可见"的最新一期数据。
  • PRef(feature, period)（pit.py:63）— 引用某个指定报告期的 PIT 值（period 如 201903 表示 2019 年 Q1）。

在字段公式里，PIT 字段用 $$（双美元）前缀区分于普通行情字段的 $：

# $$roewa_q：滚动可见的单季 ROE（加权）；对每个交易日返回当时已公布的最新值
df = D.features(
    ["SH600519"],
    ["$$roewa_q", "$$yoyni_q"],   # ROE、净利润同比
    start_time="2019-01-01",
    end_time="2020-12-31",
    freq="day",
)

底层实现依赖 utils 里的 get_period_list / read_period_data（在 data.py:26-38 已导入），把「报告期 → 公布日 → 交易日可见性」这层映射管理起来。你只管写 $$字段名，look-ahead 的防护由数据层自动完成。

小结 行情用 $field、PIT 财务用 $$field；前者当天可见，后者按公告日可见。回测涉及基本面因子时，务必走 PIT 通道，别自己 merge 财报表——那几乎必然埋下未来函数。

📝 课后练习题

1.（概念题）D.features 返回的 DataFrame 的索引结构是什么？为什么直接对多只股票的某一列画图会出错？
参考答案
返回的是双层 MultiIndex，外层 instrument、内层 datetime。直接对多只股票的一列画图，会把不同股票的时间序列首尾相连成一条锯齿曲线。应先 df.loc["SH600000"] 切出单只，或用 df["$close"].unstack(level=0) 把股票摊成列再画。
2.（代码填空）写一个字段公式，计算「当日收盘价相对过去 20 日均线的偏离百分比」。
参考答案
"$close / Mean($close, 20) - 1"。其中 $close 是原始收盘价字段（Feature），Mean(x, N) 是 Rolling 算子（ops.py:827），二者用 Div/Sub（PairOperator）组合成表达式树。
3.（改错题）有人在同一进程里跑完实验后再次调用 qlib.init(provider_uri=另一个路径)，然后抱怨第一条 D.features 又变慢了，怀疑是 bug。对吗？
参考答案
不是 bug。qlib.init() 内部会执行 H.clear()（qlib/__init__.py:56），清空内存缓存 H 的三个格子（c/i/f）。所以 re-init 后第一次取数必然要重新从磁盘读并重算，自然变慢，属于预期行为。
4.（概念题）特征在磁盘上是怎么存的？为什么这种布局利于按字段取数？
参考答案
每只股票、每个字段、每个频率各一个 .bin 文件，路径为 {code}/{field}.{freq}.bin（file_storage.py:289），文件头存起始日历下标，后接定长 float32 数组。这是列式存储：取 $close 时完全不必读取 $volume 文件，IO 最小；新增自定义因子只需补一个 .bin，无需重写整表。
5.（思考题）什么是「未来函数」？用财务数据举例说明它为什么危险，Qlib 靠什么机制避免？
参考答案
未来函数指回测中用到了当时实际拿不到的未来信息。例如公司 Q1（3 月 31 日报告期）净利润实际 4 月底才公告，若在 4 月 1 日就使用，就偷看了未来。危险在于它让回测虚高、却在实盘失效，且不报错难以察觉。Qlib 用 PIT（点数据，qlib/data/pit.py 的 P/PRef）为每期财报记录「公告可见日」，取数时对每个交易日只返回当时已公布的最新一期；字段用 $$ 前缀。
6.（进阶）滚动算子里 Slope、Rsquare 这类的核心计算是纯 Python 实现的吗？如果你在仓库根目录 import qlib 报 ImportError，最可能的原因是什么？
参考答案
不是纯 Python。它们的核心循环由 Cython 实现，从 qlib/data/_libs/rolling 和 expanding 导入（ops.py:18-19）。在仓库根目录直接 import 会因为 Cython 扩展尚未编译而 ImportError（ops.py:20-24 的报错提示正是为此）。解决办法：用 pip install . 安装，或 python setup.py build_ext --inplace 就地编译扩展，且不要在仓库根目录下 import。
