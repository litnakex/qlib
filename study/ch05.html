CHAPTER 05

模型 Model

数据准备好了，接下来该让机器「学习」了。本章讲清 Qlib 的模型抽象、训练/预测契约，并手把手教你接入一个自定义模型。

1. 从一台「榨汁机」说起

想象模型是一台榨汁机：你把水果（特征 X）倒进去，它就吐出果汁（预测值 y）。而在使用之前，你得先「调校」这台机器——告诉它哪些水果配哪种果汁，这就是训练（fit）。调校完成后，扔进任何新水果，它都能给你榨出对应的果汁，这就是预测（predict）。

在 Qlib 里，所有模型都被抽象成这样两个动作，无论底层是 LightGBM、线性回归还是深度神经网络，对外的「接口」都长得一模一样。这种统一契约让你能在配置文件里随意替换模型，而工作流的其余部分完全不用改。

提示Qlib 模型抽象定义在 qlib/model/base.py。全部三个类（BaseModel / Model / ModelFT）都在这一个文件里，代码非常短，建议先通读一遍。

2. 三层模型抽象：BaseModel / Model / ModelFT

Qlib 用三个层层递进的抽象类描述模型能力，见 qlib/model/base.py:10 起：

  BaseModel  (能预测，能像函数一样被调用)
      │  __call__ → predict
      ▼
    Model    (可学习：新增 fit)
      │  fit + predict
      ▼
   ModelFT   (可微调：新增 finetune)
      │  fit + predict + finetune

BaseModel 只规定了一个抽象方法 predict，并通过 __call__ 让模型实例可以像函数一样被调用：

class BaseModel(Serializable, metaclass=abc.ABCMeta):
    """Modeling things"""
    @abc.abstractmethod
    def predict(self, *args, **kwargs) -> object:
        """Make predictions after modeling things"""

    def __call__(self, *args, **kwargs) -> object:
        # 语法糖：pred = model(dataset) 等价于 model.predict(dataset)
        return self.predict(*args, **kwargs)

因为它继承自 Serializable，模型可以被 pickle 到磁盘、之后再加载复用——这正是 Qlib 训练完能把模型存进 recorder 的基础。

Model 在 BaseModel 之上增加了 fit(dataset, reweighter)，这是「可学习模型」的核心契约（qlib/model/base.py:22）：

class Model(BaseModel):
    """Learnable Models"""
    def fit(self, dataset: Dataset, reweighter: Reweighter):
        # 从 dataset 里取出训练数据并学习
        raise NotImplementedError()

    @abc.abstractmethod
    def predict(self, dataset: Dataset, segment="test") -> object:
        # 对 dataset 的某个 segment 做预测，默认 test 段
        raise NotImplementedError()

注意两个方法的入参都是 dataset 对象，而不是裸的 numpy 数组。模型自己负责从 dataset 里「取料」（怎么取见第 3 节）。predict 的 segment 参数决定预测哪一段数据，可以是字符串 "train"/"valid"/"test"，也可以是切片。

陷阱学到的属性不能以下划线 _ 开头！ base.py:29 的文档明确指出：fit 之后保存到 self 上的、需要被序列化的属性（如模型权重）不能命名为 self._model，否则 dump 到磁盘时会丢失。LightGBM 实现里就是用 self.model 而非 self._model。

ModelFT（Fine-tunable，base.py:81）再加一个 finetune(dataset)，用于在已训练好的模型上继续增量训练。典型场景是在线学习：先训一个初始模型存进 recorder，之后加载它、只用少量新数据 finetune 几轮：

# 训练初始模型并保存
with R.start(experiment_name="init models"):
    model.fit(dataset)
    R.save_objects(init_model=model)
    rid = R.get_recorder().id

# 加载后微调
with R.start(experiment_name="finetune model"):
    recorder = R.get_recorder(recorder_id=rid, experiment_name="init models")
    model = recorder.load_object("init_model")
    model.finetune(dataset, num_boost_round=10)

3. 真实案例：LGBModel 怎么工作

光看抽象类太空洞，我们看一个真正跑起来的模型——LightGBM，代码在 qlib/contrib/model/gbdt.py。它同时继承了 ModelFT（可微调）和 LightGBMFInt（可解释性接口）：

class LGBModel(ModelFT, LightGBMFInt):
    def __init__(self, loss="mse", early_stopping_rounds=50,
                 num_boost_round=1000, **kwargs):
        self.params = {"objective": loss, "verbosity": -1}
        self.params.update(kwargs)   # 其余 LGB 超参透传
        self.early_stopping_rounds = early_stopping_rounds
        self.num_boost_round = num_boost_round
        self.model = None            # 不带下划线，可序列化

关键在于「从 dataset 取料」这一步。看 gbdt.py:37 的 _prepare_data：

# 用 dataset.prepare 取出 feature 和 label
df = dataset.prepare(key, col_set=["feature", "label"],
                     data_key=DataHandlerLP.DK_L)  # DK_L：学习用数据
x, y = df["feature"], df["label"]
# LightGBM 需要 1 维标签，多标签会报错
y = np.squeeze(y.values)
ds = lgb.Dataset(x.values, label=y, weight=w, free_raw_data=False)

这里 dataset.prepare(...) 正是上一章数据集/处理器提供的接口。col_set 选列（feature/label），data_key 选用哪一份数据视图：DK_L（learn，训练用，可能带标签归一化）还是 DK_I（infer，推理用）。

fit（gbdt.py:57）拿到 train/valid 两个 lgb.Dataset 后调用 lgb.train，并把每轮评估指标通过 R.log_metrics 记录到实验里。predict（gbdt.py:92）则是取推理数据、返回一个带索引的 Series：

def predict(self, dataset: DatasetH, segment="test"):
    if self.model is None:
        raise ValueError("model is not fitted yet!")
    x_test = dataset.prepare(segment, col_set="feature",
                             data_key=DataHandlerLP.DK_I)  # 推理视图
    return pd.Series(self.model.predict(x_test.values), index=x_test.index)
小结一个 Qlib 模型的套路：__init__ 存超参 → fit 用 dataset.prepare(DK_L) 取训练数据学习 → predict 用 dataset.prepare(DK_I) 取推理数据、返回带索引的 Series。返回的索引（datetime, instrument）非常重要，后续策略/回测靠它对齐。

4. task_train：qrun 背后真正干活的训练单元

当你在命令行敲 qrun config.yaml 时，Qlib 到底做了什么？答案藏在 qlib/model/trainer.py 的 task_train（trainer.py:108）。它把 模型 + 数据集 + 记录器 串成一条完整流水线：

def task_train(task_config: dict, experiment_name, recorder_name=None):
    with R.start(experiment_name=experiment_name, recorder_name=recorder_name):
        _log_task_info(task_config)   # 记录配置到实验
        _exe_task(task_config)        # 真正执行训练
        return R.get_recorder()

核心逻辑在 _exe_task（trainer.py:42），把整个流程一览无余：

# 1. 用配置初始化 model 和 dataset（约束类型）
model = init_instance_by_config(task_config["model"], accept_types=Model)
dataset = init_instance_by_config(task_config["dataset"], accept_types=Dataset)
reweighter = task_config.get("reweighter", None)
# 2. 训练（auto_filter_kwargs 自动过滤 fit 不认识的参数）
auto_filter_kwargs(model.fit)(dataset, reweighter=reweighter)
R.save_objects(**{"params.pkl": model})   # 保存模型
# 3. 保存 dataset（dump_all=False：只存配置不存数据，供在线推理）
dataset.config(dump_all=False, recursive=True)
R.save_objects(**{"dataset": dataset})
# 4. 填充 <MODEL>/<DATASET> 占位符，跑 record（预测/回测/分析）
task_config = fill_placeholder(task_config, {"<MODEL>": model, "<DATASET>": dataset})
for record in task_config.get("record", []):
    r = init_instance_by_config(record, recorder=rec, ...)
    r.generate()
  config.yaml
      │  init_instance_by_config
      ▼
  ┌─────────┐   fit(dataset)   ┌──────────┐
  │  Model  │ ◀──────────────  │ Dataset  │
  └─────────┘                  └──────────┘
      │ save_objects(params.pkl / dataset)
      ▼
  ┌──────────────────────────────────────┐
  │ record: SignalRecord → 预测           │
  │         PortAnaRecord → 回测/分析      │
  └──────────────────────────────────────┘
      │  存入 Recorder / 实验
      ▼
   可复现的实验结果
提示<MODEL> 和 <DATASET> 是配置里的占位符：record 部分想引用刚训练好的 model/dataset 时写这两个字符串，fill_placeholder 会在运行时替换成真实对象。这就是 YAML 里能「引用」内存对象的魔法。

5. 接入你自己的模型

Qlib 用 init_instance_by_config 从配置构造对象，契约很简单：一个可 import 的类 + 一组 kwargs。配置长这样：

model:
  class: MyModel                          # 类名
  module_path: qlib.contrib.model.my_model # 可 import 的模块路径
  kwargs:                                  # 传给 __init__ 的参数
    lr: 0.01
    epochs: 100

所以接入自定义模型只需两步：(1) 写一个继承 Model（或 ModelFT）的类，放到能被 import 的地方（推荐 qlib/contrib/model/）；(2) 在配置里用 class/module_path/kwargs 指向它。下面是最小骨架：

import pandas as pd
from qlib.model.base import Model
from qlib.data.dataset import DatasetH
from qlib.data.dataset.handler import DataHandlerLP

class MyMeanModel(Model):
    """一个啥也没学的示例：永远预测特征均值"""
    def __init__(self, scale=1.0, **kwargs):
        self.scale = scale       # 超参存 self，不带下划线
        self.bias = None         # 学到的参数也不带下划线

    def fit(self, dataset: DatasetH, reweighter=None):
        df = dataset.prepare("train", col_set=["feature", "label"],
                             data_key=DataHandlerLP.DK_L)
        # 这里放你真正的训练逻辑
        self.bias = df["label"].mean().item()

    def predict(self, dataset: DatasetH, segment="test"):
        x = dataset.prepare(segment, col_set="feature",
                            data_key=DataHandlerLP.DK_I)
        pred = x.mean(axis=1) * self.scale + self.bias
        return pd.Series(pred, index=x.index)  # 必须带 index！
务必遵守三条铁律：① 学到的属性不能以 _ 开头（否则序列化丢失）；② predict 返回的 Series 必须带 (datetime, instrument) 索引；③ __init__ 的参数要和配置 kwargs 对应（多余的用 **kwargs 兜住，因为 auto_filter_kwargs 也会帮忙过滤）。

6. Qlib 内置模型概览

你不必事事从零造轮子。qlib/contrib/model/ 已内置一大批模型，examples/benchmarks/ 下有配套的可跑配置与基准成绩。大致分三类：

  • 树模型 / GBDT：gbdt.py（LightGBM）、xgboost.py、catboost_model.py、double_ensemble.py。工业界跑量化的主力，速度快、无需 GPU。
  • 线性模型：linear.py（LinearModel，支持 OLS / Ridge / Lasso）。简单、可解释，适合做 baseline。
  • 深度模型（PyTorch）：pytorch_lstm.py、pytorch_gru.py、pytorch_alstm.py、pytorch_transformer.py、pytorch_gats.py、pytorch_tra.py、pytorch_tabnet.py 等几十个。带 _ts 后缀的是时序版本（用 TSDatasetH 喂序列数据）。
依赖注意树模型和线性模型开箱即用；torch 系列模型需要额外安装 PyTorch 依赖（pip install qlib[rl] 或手动装 torch）。跑深度模型前请先确认 import torch 成功，且大数据集建议有 GPU。

📝 课后练习题

1. （概念题）BaseModel、Model、ModelFT 三者是什么关系？各自新增了哪个方法？
参考答案
它们是继承链：BaseModel → Model → ModelFT。BaseModel 定义了抽象方法 predict 并用 __call__ 让实例可当函数调用；Model 在其上新增 fit（可学习）；ModelFT 再新增 finetune（可微调、增量训练）。见 qlib/model/base.py。
2. （改错题）下面自定义模型有一个会导致「保存后加载丢失权重」的错误，请指出并修正。
self._weights = train(x, y)
参考答案
错误在于属性名以下划线开头。base.py:29 的文档规定：学到的、需要序列化的属性不能以 _ 开头，否则 dump 到磁盘时会被丢弃。改为 self.weights = train(x, y) 即可（对照 LGBModel 用的是 self.model）。
3. （代码理解）LGBModel 的 fit 用 DK_L 取数据，predict 用 DK_I 取数据，为什么两者不同？
参考答案
DK_L（learn）是训练用视图，可能对 label 做了归一化等处理；DK_I（infer）是推理用视图。fit 需要 feature 和 label 一起学习，故用 DK_L 并取 col_set=["feature","label"]；predict 只需 feature，故用 DK_I 且 col_set="feature"。区分两个视图是为了训练/推理阶段的数据处理逻辑可以不同。
4. （填空题）在 task_train / _exe_task 中，Qlib 用 ________(task_config["model"], accept_types=Model) 从配置构造模型对象。空格处填什么？它要求配置里提供哪三样东西？
参考答案
填 init_instance_by_config。它要求配置提供：class（类名）、module_path（可 import 的模块路径）、kwargs（传给 __init__ 的参数）。accept_types=Model 还会校验构造出的对象确实是 Model 子类。
5. （思考题）配置文件的 record 部分为什么可以写 <MODEL> / <DATASET> 这样的字符串？它们是怎么变成真实对象的？
参考答案
它们是占位符。_exe_task 训练完 model、dataset 后，调用 fill_placeholder(task_config, {"<MODEL>": model, "<DATASET>": dataset})，在运行时把 YAML 里的占位字符串替换成内存中的真实对象，从而让 record（如 SignalRecord）能引用刚训练好的模型与数据集。
6. （实践题）请写出把上文 MyMeanModel 接入 qrun 的 model 配置片段。
参考答案
model:
  class: MyMeanModel
  module_path: qlib.contrib.model.my_model
  kwargs:
    scale: 1.0
前提是把 MyMeanModel 放在 qlib/contrib/model/my_model.py（或任何能被 import 的路径）。kwargs 里的键需与 __init__ 参数对应。
