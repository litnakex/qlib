# 基准模型性能

本页列出了一批用于挖掘 alpha（超额收益）的方法。每种方法都尝试为每只股票每天给出评分/预测（例如预测股票未来的超额收益）。模型给出的评分/预测将被用作挖掘出的 alpha。投资于评分更高的股票，预期能获得更高的收益。

alpha 通过两种方式评估：
1. alpha 与未来收益之间的相关性。
1. 基于 alpha 构建投资组合并评估最终的总收益。
   - 各项指标的解释见[此处](https://qlib.readthedocs.io/en/latest/component/report.html#id4)

以下是各基准模型分别在 Qlib 的 `Alpha360` 和 `Alpha158` 数据集上、使用中国 A 股 & CSI300 数据运行的结果。每个指标的数值是基于 20 次不同随机种子运行计算出的均值和标准差。

下面展示的数字体现了每个模型完整 `workflow` 的表现。我们将在不久的将来更新 `workflow` 以及模型，以取得更好的结果。

# 贡献

非常欢迎你为新模型做出贡献！

如果你想贡献新模型，可以按下面的步骤操作。
1. 为你的模型创建一个文件夹
2. 该文件夹包含以下内容（可参考[这个示例](https://github.com/microsoft/qlib/tree/main/examples/benchmarks/TCTS)）。
    - `requirements.txt`：所需依赖。
    - `README.md`：对你的模型的简要介绍
    - `workflow_config_<模型名>_<数据集>.yaml`：一份可被 `qrun` 读取的配置。鼓励你在所有数据集上运行你的模型。
3. 你可以把你的模型作为一个模块集成[到该文件夹中](https://github.com/microsoft/qlib/tree/main/qlib/contrib/model)。
4. 请在上面的**基准表格**中更新你的结果，例如 [Alpha360](#alpha158-dataset)、[Alpha158](#alpha158-dataset)（每个指标的数值是基于 **20 次运行**、使用不同随机种子计算出的均值和标准差。你可以通过 Qlib 提供的自动化[脚本](https://github.com/microsoft/qlib/blob/main/examples/run_all_model.py)完成上述操作，并在 .md 文件中得到最终结果。如果你没有足够的计算资源，可以在 PR 中寻求帮助）。
5. 更新首页的[新闻列表](https://github.com/microsoft/qlib#newspaper-whats-new----sparkling_heart)和[模型列表](https://github.com/microsoft/qlib#quant-model-paper-zoo)中的信息。

最后，你可以提交 PR 供审阅。（[这是一个示例](https://github.com/microsoft/qlib/pull/1040)）


# 常见问题

问：名称为 `*.py` 和 `*_ts.py` 的模型有什么区别？

答：名称为 `*_ts.py` 的模型是为 `TSDatasetH` 设计的（`TSDatasetH` 会从表格数据自动创建时间序列）。名称为 `*.py` 的模型是为 `DatasetH` 设计的（`DatasetH` 通常用于表格数据。但如果列之间具有时间序列关系，用户仍可在表格数据集上应用时间序列模型）。
